# Collection Merchandising Skill

Use this skill when building or improving collection pages, list-collections pages, and grid-based browsing experiences.

## Objective

Turn category and collection views into guided shopping experiences that improve discovery, trust, and conversion.

## Core Principles

- Browsing should feel curated, not overwhelming
- Product discovery should be faster than default Shopify
- Merchandising should support buying intent
- Layout should feel premium, not marketplace-noisy

## Collection Page Priorities

1. Clear context
2. Useful filters and sorting
3. Strong product cards
4. Trust reinforcement
5. Merchandising support
6. Graceful empty states

## Recommended Collection Structure

### Top Section

Include:

- collection title
- short supporting description if available
- product count
- optional collection image / visual context
- optional trust row

### Controls

Include:

- sort
- filters
- active filter visibility
- clear all action when filters exist
- mobile filter drawer when needed

### Product Grid

Product cards should support:

- stable image ratio
- title
- product type/category
- price
- compare-at if valid
- rating if available
- badge support
- clear CTA affordance

### Merchandising Blocks

Useful modules:

- Top picks this week
- Clinic favorites
- Best for first-time pet parents
- Shop by pet type
- Trust strip
- Popular products row

## List Collections Page Priorities

- Turn a plain directory into a shopping guide
- Make category cards feel intentional
- Support category trust and relevance
- Add merchandising below the grid

Useful modules:

- hero with category context
- quick chips
- featured collections
- popular products
- trust strip

## CRO Rules

- No fake urgency
- No fake ratings
- No fake popularity metrics
- Editorial labels are allowed if clearly non-numeric
- Trust strips and merchandising language must feel believable and premium

## UX Rules

- Filters should be easy to scan
- Sort must not feel buried
- Mobile browsing should not feel cramped
- Empty results should help users recover, not dead-end

## Edge Cases

Always handle:

- no collection image
- no description
- few products
- many products
- empty results
- missing ratings
- long titles
- sold out items
- products with multiple badges

## Output Expectations

When using this skill:

1. Identify what makes the page feel flat or default
2. Improve top-of-page merchandising
3. Improve filter/sort usability
4. Improve product grid quality
5. Add trust / merchandising modules only where they support browsing
