# CRO Patterns Skill

Use this skill when building or reviewing Shopify storefront components that should improve conversion rate, average order value, trust, or product discovery.

## Objective

Apply proven CRO patterns in a way that feels premium, honest, and aligned with the FOOFASH design system.

## Core Principles

- Conversion should come from clarity, trust, and relevance
- Never use fake urgency or fake social proof
- CRO elements must feel integrated, not spammy
- Mobile conversion matters as much as desktop
- Premium brands should persuade quietly, not aggressively

## Approved CRO Patterns

### 1. Primary CTA Hierarchy

Use a single dominant action per section or view.

Good examples:

- PDP: Add to Cart
- Collection: Browse / Quick Add
- Homepage hero: Shop Best Sellers

Rules:

- CTA must be visually obvious
- Secondary actions must not compete
- Use spacing and size before using louder colors

### 2. Trust Signals

Use trust near decision points.

Allowed trust elements:

- Vet Approved
- Free shipping threshold
- Easy returns
- Rating stars from real metafields
- Review count from real data
- Professional/clinic language where appropriate

Placement:

- Under price on PDP
- Near CTA
- In trust strips on homepage/collection pages
- Inside cart drawer when useful

### 3. Honest Urgency

Urgency is only allowed when backed by real data.

Allowed:

- Only X left (tracked inventory only, quantity > 0, threshold <= 5)
- Order before 3PM for same-day dispatch (only if configured and true)
- Limited-time sale only if actual compare-at/sale exists

Forbidden:

- Fake countdowns
- Fake stock warnings
- Fake “12 people viewing”
- Fake review stats

### 4. Social Proof

Use only real or clearly labeled editorial proof.

Allowed:

- Rating stars from metafields
- Review counts from metafields
- “Clinic Pick”
- “Popular Choice”
- “Pet Parent Favorite” only when clearly used as merchandising language, not numerical proof

Rules:

- Do not fabricate stars
- Do not fabricate counts
- If data is missing, hide the block or use a non-numeric editorial label

### 5. AOV Patterns

Use subtle, relevant upsells.

Good placements:

- PDP: Complete the Kit
- Cart drawer: You may also need
- Homepage: Featured favorites / bundles
- Collection page: merchandising blocks

Rules:

- Upsells must be relevant
- Avoid showing too many offers at once
- Prioritize 1–3 strong recommendations over clutter

### 6. Browsing Support

Help users discover products faster.

Useful patterns:

- Shop by category
- Pet type chips
- Best seller rows
- Collection trust strips
- Predictive search
- Quick view
- Sticky sort/filter controls on mobile if helpful

### 7. Reduced Friction

Make buying feel easy.

Patterns:

- Sticky ATC on PDP
- Cart drawer instead of full redirect
- Pill variant selectors where appropriate
- Clear quantity controls
- Accordion content for long info
- Quick view on collection grids

## Page-Specific Guidance

### Homepage

Prioritize:

- value proposition
- trust
- featured products
- category navigation
- testimonials
- best sellers / popular products

### Collection Page

Prioritize:

- clear collection context
- refined filters/sort
- product grid quality
- merchandising support
- trust strip
- empty-state recovery

### PDP

Prioritize:

- title
- price
- rating/trust
- variants
- CTA
- shipping/returns reassurance
- sticky ATC
- related products / bundles

### Cart

Prioritize:

- line-item clarity
- easy edits
- threshold cues
- relevant upsell
- friction-free checkout path

## Output Expectations

When applying this skill:

1. Identify the page’s main conversion goal
2. Identify friction or missing trust
3. Add only the most relevant CRO elements
4. Keep the design premium and uncluttered
5. Ensure all signals are backed by real data or clearly editorial labels
