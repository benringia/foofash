# Shopify Theme Dev Environment — Design Spec

**Date:** 2026-04-02  
**Project:** foofash  
**Status:** Approved

---

## Context

Initialize a production-ready Shopify Online Store 2.0 theme development environment optimized for conversion rate performance. The stack is locked: Liquid (OS 2.0), Tailwind CSS (JIT), Vanilla JS (ES Modules), and Vite as the build tool.

The project directory is currently empty. This spec covers the full scaffold from zero.

---

## Stack (Locked)

| Layer | Technology |
|-------|------------|
| Templates | Shopify Liquid (OS 2.0) |
| Styles | Tailwind CSS v3 (JIT) |
| Scripts | Vanilla JS — ES Modules only |
| Build tool | Vite v5 |
| Shopify API | Shopify AJAX Cart API (`/cart.js`, `/cart/add.js`, `/cart/change.js`) |
| Dev orchestration | `concurrently` (Vite + Shopify CLI) |

**Separation contract:** `/src` is development source only. `/shopify` is the theme output only. Nothing crosses the boundary except through the Vite build pipeline.

## Build Pipeline

### Dev workflow

```
npm run dev
  ├── vite                  # HMR dev server on localhost:5173
  └── shopify theme dev     # Shopify CLI proxies store, watches /shopify/**
```

Both processes start concurrently with a single command.

### Production build

```
npm run build
  ├── vite build
  │     └── outputs to shopify/assets/
  │           ├── main.[hash].js
  │           ├── main.[hash].css
  │           └── .vite/manifest.json
  └── node scripts/generate-liquid-assets.mjs
        └── reads manifest.json
        └── writes shopify/snippets/vite-assets.liquid
```

### `package.json` scripts

```json
{
  "type": "module",
  "scripts": {
    "dev": "concurrently 'vite' 'shopify theme dev'",
    "build": "vite build && node scripts/generate-liquid-assets.mjs",
    "push": "shopify theme push"
  }
}
```

### Dependencies

```json
{
  "devDependencies": {
    "vite": "^5",
    "tailwindcss": "^3",
    "postcss": "^8",
    "autoprefixer": "^10",
    "concurrently": "^9"
  }
}
```

No runtime dependencies. No frameworks. No jQuery.

## Vite Configuration

```js
// vite.config.js
export default {
  root: 'src',
  build: {
    outDir: '../shopify/assets',
    emptyOutDir: false,       // Never wipe /assets — it contains non-built theme files
    manifest: true,
    rollupOptions: {
      input: 'scripts/main.js'   // Relative to root ('src'). CSS imported inside main.js.
    }
  }
}
```

**`emptyOutDir: false`** is required. Vite's output directory is also Shopify's `/assets`, which holds images, fonts, and other theme files not managed by Vite. Wiping on every build would break the theme.

**Single entry point:** `main.js` imports `../styles/main.css`. Vite extracts CSS automatically into `main.[hash].css`. No separate CSS Rollup entry needed — that approach would require paths relative to `root`, which is fragile and non-standard.

## Manifest → Liquid Asset Generation

### Problem

Shopify Liquid cannot read `manifest.json` at runtime. Hashed filenames are unknown until after `vite build`.

### Solution

`scripts/generate-liquid-assets.mjs` runs after `vite build` as part of `npm run build`.

**Logic:**
1. Read `shopify/assets/.vite/manifest.json`
2. Find the JS entry (`scripts/main.js`) — its `file` is `main.[hash].js`, its `css` array contains `main.[hash].css`
3. Write `shopify/snippets/vite-assets.liquid`
4. On missing manifest: log a clear error, exit with code 1, leave existing snippet untouched

**Output (`shopify/snippets/vite-assets.liquid`):**
```liquid
{% comment %}Auto-generated by scripts/generate-liquid-assets.mjs — do not edit{% endcomment %}
{% assign vite_css = 'main.abc123.css' | asset_url %}
{% assign vite_js = 'main.abc123.js' | asset_url %}
<link rel="stylesheet" href="{{ vite_css }}" type="text/css">
<script src="{{ vite_js }}" type="module" defer></script>
```

This file is committed to source control so the theme is always deployable, even before a fresh build.

## Shopify Liquid Integration

### `shopify/layout/theme.liquid`

```liquid

  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">

      {{ content_for_layout }}
    </main>
  </body>

```

### `shopify/templates/index.json`

OS 2.0 JSON template. Sections declared here, not in `theme.liquid`.

```json
{
  "sections": {
    "main-hero": {
      "type": "main-hero",
      "settings": {}
    }
  },
  "order": ["main-hero"]
}
```

### `shopify/sections/main-hero.liquid`

Minimal section with schema. Schema `settings` block prevents Liquid errors when settings are missing.

```liquid
<section class="main-hero">
  <h1>{{ section.settings.heading }}</h1>
</section>

{% schema %}
{
  "name": "Hero",
  "settings": [
    { "type": "text", "id": "heading", "label": "Heading", "default": "Welcome" }
  ],
  "presets": [{ "name": "Hero" }]
}
{% endschema %}
```

## JavaScript Architecture

### Entry Point (`src/scripts/main.js`)

```js
import '../styles/main.css'   // Vite extracts this into main.[hash].css at build time
import { initCart } from './cart.js'
import { initProduct } from './product.js'

document.addEventListener('DOMContentLoaded', () => {
  initCart()
  initProduct()
})
```

### Module contracts

| File | Exported function | Responsibility |
|------|-------------------|----------------|
| `cart.js` | `initCart()` | Cart drawer, quantity updates, AJAX cart API |
| `product.js` | `initProduct()` | Variant switching, option selection, add-to-cart |
| `ui/modal.js` | `initModal()` | Open/close, focus trap, ESC key |
| `ui/accordion.js` | `initAccordion()` | Toggle expand/collapse |

### Rules

- No `window.*` assignments — no global pollution
- No framework imports — Vanilla JS ESM only
- `init*()` functions are no-ops if their target DOM elements are absent — silent fail, no errors thrown
- All Shopify API calls use `fetch` with `Content-Type: application/json`
- `const` by default, `let` only when reassignment needed

## Edge Case Handling

| Scenario | Handling |
|----------|----------|
| Missing Vite manifest | `generate-liquid-assets.mjs` exits 1 with error; existing snippet preserved |
| Build failure | `/shopify/assets` unchanged (`emptyOutDir: false`); theme stays deployable |
| Empty sections | Schema `settings` with defaults prevents Liquid errors |
| Missing JS module target elements | `init*()` guards with `if (!el) return` — silent fail |
| Tailwind purging needed classes | Add to `safelist` in `tailwind.config.js` as patterns emerge |
| Shopify CDN caching | Resolved by hashed filenames in every build |
| Dev vs production path mismatch | Both environments use `asset_url` filter — consistent paths |
